# 건강운동관리사 모바일 퀴즈/오답노트 플랜

## 목표

아내가 휴대폰에서 하루 10문제를 바로 풀 수 있는 HTML 퀴즈를 만들고, 풀이 결과를 바탕으로 오답노트를 누적 생성한다.

최종 사용 흐름:

1. 원본 기출 PDF를 로컬에 보관한다.
2. 하루치 문제 10개를 JSON으로 만든다.
3. JSON을 모바일 HTML 퀴즈로 변환한다.
4. HTML 파일을 카톡/파일로 전달한다.
5. 아내가 문제를 풀고 결과를 복사하거나 제출한다.
6. 결과를 누적해서 과목별/개념별 오답노트를 만든다.

## 현재 확보한 원본 자료

공식 KSPO 체육지도자 자격검정 및 연수 게시판의 건강운동관리사 필기시험 문제지/정답 자료를 내려받아 보관한다.

- 위치: `materials/raw/kspo/`
- 범위: 2019~2025년
- 포함: 문제지, 정답가안, 최종정답
- 목록: `materials/raw/kspo/manifest.json`

## 시험 임박 출제 전략

시험 임박기에는 하루 한 과목 순환을 중단하고 전과목 혼합 10문항으로 운영한다.

기본 배분:

| 과목 | 문항 수 |
|---|---:|
| 운동생리학 | 2 |
| 건강·체력평가 | 2 |
| 운동처방론 | 1 |
| 운동부하검사 | 1 |
| 운동상해 | 1 |
| 기능해부학 | 1 |
| 병태생리학 | 1 |
| 스포츠심리학 | 1 |

오답 데이터가 쌓이면 최근 7일 오답 과목과 반복 오답 topic을 우선 반영한다. 단, 8과목 모두 최소 1문항 이상 포함하고 총 문항 수는 10문항으로 고정한다.

정책 파일:

- `config/daily-selection-policy.json`

### 단기 중복 방지

문항 중복은 세 단계로 막는다.

1. 같은 `questionId`는 최근 14일 안에 재출제하지 않는다.
2. A형/B형만 다른 동일 문항은 같은 문제로 본다.
3. 같은 `topic`은 최근 3일 안에 반복하지 않는다. 단, 반복 오답 복습일 때는 같은 원문 문항 대신 같은 topic의 유사 문항을 낸다.

추가 품질 기준:

- 같은 정답 번호가 10문항 중 4개를 넘지 않게 한다.
- 한 과목은 하루 최대 4문항을 넘지 않게 한다.
- 같은 topic은 하루 최대 2문항으로 제한한다.
- 계산 문제는 하루 최대 2문항으로 제한한다.
- 긴 보기 조합 문제를 연속 배치하지 않는다.
- 금기증, 수치 기준, 계산, 용어 구분, 개념 판단 유형을 섞는다.

중복 방지 외에 같이 신경쓸 부분:

- **정답 번호 편향**: ①이나 ④가 지나치게 많으면 찍기 습관이 생긴다.
- **난도 피로**: 고오답 문제만 20개 넣으면 완주율이 떨어진다.
- **과목 편중**: 약점 과목을 보강하되 8과목 감각은 매일 유지한다.
- **오답 원인 분류**: 단순 오답 수보다 `trap`을 누적해야 다음 출제가 좋아진다.
- **해설 길이**: 모바일에서는 문항당 3줄 이내가 유지되어야 한다.
- **출처 연결**: 문제마다 원본 PDF, 연도, 과목, 문항번호를 유지한다.
- **복습 간격**: 틀린 바로 다음 날 같은 원문을 다시 내기보다 2~3일 뒤 유사 문항을 낸다.
- **미응답 처리**: 틀린 문제와 별도로 시간이 부족해 못 푼 문제를 기록한다.

## 1단계: 서버 없는 HTML MVP

처음 버전은 서버 없이 HTML 파일 하나로 동작하게 만든다.

### 기능

- 모바일 세로 화면 최적화
- 하루 10문제
- 문제와 보기 먼저 표시
- 선택 즉시 정답/오답 표시
- 오답 시 해설 자동 표시
- 정답이어도 해설 보기 가능
- 최종 점수 표시
- 틀린 문제만 모아 보기
- 오답 결과 복사 버튼

### 장점

- 운영비 없음
- 로그인 없음
- DB 없음
- 카톡 파일 전송 가능
- 휴대폰 브라우저에서 바로 실행 가능

### 한계

- 결과가 자동으로 남편 쪽 컴퓨터에 저장되지는 않는다.
- 아내가 결과 복사 버튼을 눌러 카톡으로 보내는 절차가 필요하다.
- 아이폰 카톡 미리보기와 Files 앱 Quick Look은 로컬 HTML의 JavaScript 실행을 막을 수 있다.
- 아이폰에서 안정적으로 쓰려면 Safari에서 열 수 있는 웹 링크 방식이 파일 직접 전송보다 낫다.

### 아이폰 전달 방식

아이폰에서는 HTML 파일을 카톡 첨부로 직접 열면 정상 동작하지 않을 수 있다. 카톡이나 Files 앱이 HTML을 브라우저 앱이 아니라 미리보기 문서로 처리하기 때문이다.

권장 순서:

1. 가장 안정적: GitHub Pages, Cloudflare Pages, Netlify 같은 정적 웹 링크로 열기
2. 차선: iCloud Drive에 저장 후 Safari에서 열기
3. 비권장: 카톡 첨부파일 미리보기에서 바로 풀기

현재 HTML은 외부 라이브러리 없이 한 파일로 동작하므로, 정적 웹 링크로 올리면 아이폰 Safari에서도 같은 화면을 사용할 수 있다.

## 2단계: 오답노트 반자동화

아내가 보낸 결과 텍스트를 `results/attempts.csv` 또는 `results/attempts.jsonl`에 누적한다.

### 결과 데이터 예시

```json
{
  "date": "2026-05-18",
  "quizId": "2026-05-18-exercise-physiology",
  "score": 7,
  "total": 10,
  "wrong": [
    {
      "questionId": "2023-exphys-q02",
      "subject": "운동생리학",
      "topic": "젖산역치",
      "selected": 3,
      "answer": 2
    }
  ]
}
```

### 오답노트 생성 기준

- 같은 `topic`을 2회 이상 틀리면 반복 오답으로 표시
- 최근 7일 내 틀린 문제를 우선 복습 대상으로 표시
- 과목별 정답률을 계산한다.
- 반복 오답 개념은 다음 퀴즈 후보에 우선 배정한다.

## 3단계: 자동 수집 서버

반자동 방식이 불편하면 서버를 붙인다.

### 후보

- Google Apps Script + Google Sheets
- Supabase
- 작은 Node 서버

### 서버를 붙일 때 필요한 결정

- 결과 저장 위치
- 공개 URL 여부
- 제출 인증 방식
- 개인정보 저장 범위
- 백업 방식

## 추천 구현 순서

1. 원본 PDF 보관 완료
2. 문제 JSON 형식 확정 완료
3. 검증된 문제 JSON을 `data/question-bank/seed.jsonl` 문제은행으로 누적
4. `scripts/generate_daily_quiz.py`로 하루치 JSON 자동 생성
5. `scripts/generate_quiz_html.py`로 모바일 HTML 변환
6. 모바일 화면 테스트
7. 오답 결과 복사 기능 추가 완료
8. 결과 텍스트를 누적 데이터로 변환하는 스크립트 작성 완료
9. 누적 오답노트 HTML/Markdown 생성
10. 필요하면 서버 제출 기능 추가

## 매일 생성 방법

검증된 문제은행이 충분히 쌓이면 아래 명령 하나로 하루치 JSON과 전달용 HTML을 만든다.

```bash
python3 scripts/generate_daily_quiz.py --date 2026-05-19 --html
```

생성 위치:

- 퀴즈 JSON: `data/quizzes/YYYY-MM-DD-daily.json`
- 전달 HTML: `~/Desktop/건강운동관리사/건강운동관리사_YYYY-MM-DD.html`

현재는 2023년 A형 문제은행으로 8과목/10문항 운영본 생성이 가능하다.

아이폰 웹 링크용 정적 사이트를 만들려면 아래 명령을 추가로 실행한다.

```bash
python3 scripts/build_static_site.py
```

생성 위치:

- 정적 사이트 폴더: `~/Desktop/건강운동관리사_web`
- 목록 페이지: `~/Desktop/건강운동관리사_web/index.html`
- 퀴즈 파일: `~/Desktop/건강운동관리사_web/quizzes/`

이 폴더만 GitHub Pages, Cloudflare Pages, Netlify 같은 정적 호스팅에 올린다. 프로젝트 전체를 올리면 원본 PDF까지 포함될 수 있으므로 배포 대상에서 제외한다.

## 완료 조건

- `output/YYYY-MM-DD-quiz.html` 하나만 보내도 휴대폰에서 풀 수 있다.
- 정답/해설이 처음부터 노출되지 않는다.
- 선택 후 정답/오답이 명확히 표시된다.
- 최종 화면에서 점수와 오답 결과 복사 영역을 확인할 수 있다.
- 오답 결과를 복사해서 남편에게 보낼 수 있다.
- 받은 결과를 누적 오답노트로 변환할 수 있다.

## 주의 사항

- 유료 교재, 로그인 자료, 앱 내부 자료는 우회 수집하지 않는다.
- 공식 원본 PDF는 로컬 보관용으로 둔다.
- 실제 퀴즈 데이터에는 출처와 원문 위치를 남긴다.
- 공식 오답률이 없으면 "고오답 추정"으로만 표기한다.
- API/LLM 자동 생성 단계로 가면 비용이 생길 수 있다.
